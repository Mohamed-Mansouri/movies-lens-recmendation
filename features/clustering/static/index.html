How Clustering Works

Features

Each user is represented by 42 features before clustering:

  • user_mean, user_count, user_std — rating behaviour
  • 19 genre watch-share values — fraction of total ratings per genre
  • 20 SVD latent dimensions — collaborative taste signal

All features are standardised with StandardScaler before clustering.

K-Means

Partitions users into K non-overlapping clusters by minimising inertia (within-cluster sum of squared distances to centroid).

  • K is searched over 2–10
  • Best K is chosen by the highest silhouette score
  • Each user is assigned exactly one cluster

Silhouette score: how similar a user is to their own cluster vs. the nearest other cluster. Range −1 to +1 — higher is better.

DBSCAN

Density-Based Spatial Clustering of Applications with Noise. Groups dense regions; sparse users become noise (−1).

  • eps auto-tuned: 90th percentile of 5-nearest-neighbour distances
  • min_samples = 5
  • Does not require specifying K upfront
  • Noise users (−1) do not belong to any cluster
